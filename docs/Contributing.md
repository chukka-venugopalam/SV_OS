# Contributing to SV-OS

## Getting Started

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Install dependencies: `pnpm install`
4. Make your changes following our [Coding Standards](./CodingStandards.md)
5. Run quality checks: `pnpm lint && pnpm typecheck && pnpm test`
6. Commit using conventional commits: `feat: add knowledge graph zoom`
7. Push and open a Pull Request

## Pull Request Process

1. Fill out the PR template completely
2. Ensure all CI checks pass
3. Request review from the appropriate team
4. Address all review comments
5. Squash commits before merge

## What We Look For

- Clean, maintainable code
- Proper typing (no `any`)
- Test coverage for new features
- Updated documentation
- Updated `.ai/` tracking files for architectural changes
- No regression in existing functionality

## Code of Conduct

Be respectful, constructive, and collaborative.
